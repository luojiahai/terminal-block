# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`terminal-block` is a Vue 3 component library published as an npm package. It renders macOS-style terminal windows with support for different "app" configurations (bash, claude-code). Built with Vite in library mode; `vue` is a peer dependency and is externalized from the bundle.

## Commands

```sh
pnpm install          # install dependencies
pnpm run dev          # dev server with demo App.vue
pnpm run build        # type-check + build library to dist/
pnpm run test:unit    # run unit tests (jsdom, Vitest)
pnpm run test:e2e     # run Playwright e2e tests (requires build first)
pnpm run lint         # oxlint then eslint (both with --fix)
pnpm run format       # prettier on src/
```

Run a single unit test file:
```sh
pnpm run test:unit src/components/__tests__/TerminalBlock.spec.ts
```

## Architecture

### Public API (`src/index.ts`)
Everything exported from `src/index.ts` is part of the public package API. Components, theme utilities, and the iTerm2 parser are all re-exported here.

### Title key (`src/components/TerminalBlock.vue`)
`TERMINAL_BLOCK_TITLE_KEY` is declared and exported from `TerminalBlock.vue` via a dual `<script>` + `<script setup>` block. `TerminalBlock` provides it; each app wrapper (`ClaudeCode.vue`, `Bash.vue`) injects it via `watchEffect` to keep the titlebar label in sync with its `title` prop.

`ClaudeCode` and `Bash` are exported as compound components via `Object.assign(Component, { Input, Output })`, enabling `<ClaudeCode.Input>` usage.

### Theme system (`src/themes/`)
`ThemeTokens` is a flat record of CSS custom property values (colors only). `TerminalBlock` merges `defaultTheme` → user's `theme` prop, then injects them as inline `--terminal-block-*` CSS vars on the root element. `ClaudeCode` overrides `--terminal-block-accent` to `#d97757` in its own scoped CSS, independently of the theme system.

`resolveTheme` accepts a named theme string or a partial `ThemeTokens` object. `parseItermColors` maps an iTerm2 `.itermcolors` XML string to a `Partial<ThemeTokens>`.

### Turn components (`src/components/`)
Each app has its own dedicated `Input.vue` and `Output.vue` with glyphs and colors hardcoded in scoped CSS — they don't use injection. `ClaudeCode/Input` renders `❯`, `ClaudeCode/Output` renders `⏺`, `Bash/Input` renders `$`, `Bash/Output` has no glyph.
- `Thinking` — shows "Verb..." when `done=false`, "Thought for <slot>" when `done=true`

### Build output
Vite library mode produces `dist/terminal-block.js` (ESM) and `dist/terminal-block.umd.cjs` (CJS). Type declarations are generated by `vite-plugin-dts` from `src/index.ts` and its dependencies. `src/App.vue` and `src/main.ts` are excluded from the type output.
